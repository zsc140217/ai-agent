# 微信云托管部署完整教程（超详细版）

## 🎯 核心概念：你不需要自己构建！

**重要：** 你只需要上传源代码，Docker 会自动帮你编译打包！

---

## 📦 第一步：打包项目源代码

### 方法A：使用自动脚本（最简单）⭐

1. 双击运行项目根目录下的 `打包脚本.bat`
2. 等待几秒，会生成 `backend-deploy.zip`
3. 完成！

### 方法B：手动打包

1. 打开项目文件夹：`e:\Desktop\ai-agent\jblmj-ai-agent-master`

2. **选择以下文件**（按住 Ctrl 多选）：
   ```
   ✅ src 文件夹
   ✅ pom.xml
   ✅ Dockerfile
   ✅ mvnw
   ✅ mvnw.cmd
   ✅ .mvn 文件夹
   ```

3. **不要选择**：
   ```
   ❌ target 文件夹（编译产物，不需要）
   ❌ .git 文件夹（版本控制，不需要）
   ❌ jblmj-ai-agent-frontend 文件夹（前端单独部署）
   ❌ yu-image-search-mcp-server 文件夹（MCP服务，可选）
   ❌ images 文件夹（文档图片，不需要）
   ```

4. 右键 → **发送到** → **压缩(zipped)文件夹**

5. 命名为：`backend-deploy.zip`

## 🚀 第三步：创建并部署服务

### 3.1 创建服务

1. 在左侧菜单，点击 **"云托管"** → **"服务管理"**

2. 点击右上角 **"新建服务"**

3. 填写服务信息：
   ```
   服务名称：yu-ai-agent-backend
   部署方式：Dockerfile
   ```

4. 点击 **"下一步"**

### 3.2 上传代码

1. 代码来源：选择 **"本地代码上传"**

2. 点击 **"上传代码包"**，选择刚才打包的 `backend-deploy.zip`

3. 等待上传完成（根据网速，可能需要 1-3 分钟）

### 3.3 配置容器

填写以下配置：

```
📌 基础配置：
- Dockerfile 路径：Dockerfile（默认就是这个）
- 监听端口：80
- 流量策略：100%

📌 资源配置：
- CPU：0.5 核
- 内存：1 GB
- 实例数量：1

📌 高级配置（可选）：
- 构建超时：1800 秒（30分钟）
- 启动超时：300 秒（5分钟）
```

### 3.4 开始部署

1. 点击 **"开始部署"**

2. 你会看到构建日志实时滚动，大概会经历：
   ```
   ⏳ 正在上传代码...
   ⏳ 正在构建镜像...
   ⏳ 正在下载 Maven 依赖...（这步最慢，5-10分钟）
   ⏳ 正在编译项目...
   ⏳ 正在打包 jar...
   ⏳ 正在启动容器...
   ✅ 部署成功！
   ```

3. **整个过程大约需要 10-15 分钟**，请耐心等待

## 🔍 Dockerfile 工作原理详解

你的 Dockerfile 分为两个阶段：

### 阶段1：构建阶段（自动编译）
```dockerfile
FROM maven:3.9-amazoncorretto-21 AS builder
# 使用带 Maven 和 Java 21 的镜像

WORKDIR /app
# 设置工作目录

COPY pom.xml .
COPY src ./src
# 复制源代码到容器

RUN mvn clean package -DskipTests
# 🔥 自动执行 Maven 打包
# 这一步会：
# 1. 下载所有依赖（Spring Boot、Spring AI 等）
# 2. 编译 Java 代码
# 3. 生成 jar 包到 target 目录
```

### 阶段2：运行阶段（精简镜像）
```dockerfile
FROM amazoncorretto:21-alpine
# 使用更小的 Java 运行环境（只有 100MB 左右）

WORKDIR /app

COPY --from=builder /app/target/yu-ai-agent-0.0.1-SNAPSHOT.jar app.jar
# 从阶段1复制编译好的 jar 包

EXPOSE 80
# 声明容器使用 80 端口

CMD ["java", "-jar", "app.jar", "--spring.profiles.active=prod"]
# 启动应用，使用 prod 配置文件
```

**为什么用两阶段构建？**
- 阶段1 需要 Maven（很大，约 500MB）
- 阶段2 只需要 Java 运行环境（很小，约 100MB）
- 最终镜像只包含阶段2，体积小、启动快

## ⚠️ 常见问题排查

### Q1: 构建失败："下载依赖超时"

**原因：** Maven 下载依赖太慢

**解决方案：** 添加国内镜像源

在项目根目录创建 `.mvn/maven.config` 文件：
```
-Dmaven.repo.local=/app/.m2/repository
```

或者修改 `pom.xml`，在 `<repositories>` 前添加：
```xml
<repositories>
    <repository>
        <id>aliyun</id>
        <url>https://maven.aliyun.com/repository/public</url>
    </repository>
</repositories>
```

### Q2: 部署成功但访问 404

**检查清单：**
1. 确认访问地址后面加了 `/api`
2. 检查 `application-prod.yml` 里的 `context-path: /api`
3. 查看容器日志，确认应用启动成功

### Q3: 数据库连接失败

**原因：** 阿里云 RDS 白名单限制

**解决方案：**
1. 登录阿里云 RDS 控制台
2. 找到你的数据库实例
3. 点击 **"数据安全性"** → **"白名单设置"**
4. 添加：`0.0.0.0/0`（允许所有 IP，测试用）
5. 生产环境建议只添加微信云托管的出口 IP

### Q4: 内存不足，容器重启

**解决方案：**
1. 在服务配置里，把内存改为 **2GB**
2. 或者在 Dockerfile 的 CMD 行添加内存限制：
   ```dockerfile
   CMD ["java", "-Xmx768m", "-jar", "app.jar", "--spring.profiles.active=prod"]
   ```

### Q5: 日志里显示 "端口已被占用"

**解决方案：**
确认 `application-prod.yml` 里的端口是 `80`，不是 `8123`
