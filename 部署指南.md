## 🚀 部署步骤（5步完成）

### 第1步：登录微信云托管

1. 打开浏览器，访问：https://cloud.weixin.qq.com/
2. 使用微信扫码登录
3. 如果是第一次使用，需要：
   - 创建一个环境（选择"按量计费"即可）
   - 环境名称随便起，比如：`ai-agent-prod`

### 第2步：创建服务

1. 在微信云托管控制台，点击左侧菜单 **"服务管理"**
2. 点击 **"新建服务"**
3. 填写信息：
   - 服务名称：`yu-ai-agent`
   - 部署方式：选择 **"Dockerfile"**
   - 代码来源：选择 **"代码包上传"**

### 第3步：打包代码

在你的项目根目录（就是有 `pom.xml` 的那个文件夹），把整个项目打包成 zip：

**Windows 操作：**
1. 选中项目文件夹里的所有文件（不是文件夹本身）
2. 右键 → 发送到 → 压缩(zipped)文件夹
3. 命名为：`yu-ai-agent.zip`

**注意：** 不要包含这些文件夹（如果有的话）：
- `target/`
- `node_modules/`
- `.git/`

### 第4步：上传并部署

1. 在微信云托管的"新建服务"页面，上传刚才打包的 `yu-ai-agent.zip`
2. 配置容器信息：
   - **端口**：填 `80`
   - **CPU**：选择 `0.5核`（够用了）
   - **内存**：选择 `1GB`（最低配置）
   - **实例数量**：填 `1`

3. 点击 **"开始部署"**

4. 等待部署完成（大约 5-10 分钟）
   - 你会看到构建日志滚动
   - 等到显示 **"部署成功"** 就好了

### 第5步：测试访问

1. 部署成功后，微信云托管会给你一个访问地址，类似：
   ```
   https://你的服务名-xxx.service.tcloudbase.com
   ```

2. 在浏览器访问：
   ```
   https://你的服务名-xxx.service.tcloudbase.com/api/health
   ```
   
3. 如果看到返回 `OK` 或类似信息，说明部署成功！

4. 访问 API 文档：
   ```
   https://你的服务名-xxx.service.tcloudbase.com/api/doc.html
   ```

## ⚠️ 常见问题

### Q1: 部署失败，显示"构建超时"
**解决方案：** 
- 第一次构建需要下载很多依赖，可能需要 10-15 分钟
- 在微信云托管设置里，把"构建超时时间"改为 30 分钟

### Q2: 部署成功但访问不了
**检查清单：**
1. 确认端口设置为 `80`
2. 确认数据库地址能从外网访问
3. 查看容器日志，看有没有报错

### Q3: 数据库连接失败
**解决方案：**
- 检查阿里云 RDS 的白名单设置
- 需要把微信云托管的出口 IP 加入白名单
- 或者直接设置为 `0.0.0.0/0`（允许所有 IP，测试用）

### Q4: 内存不够用
**解决方案：**
- 在服务配置里，把内存调整为 2GB
- 或者在 Dockerfile 里添加 JVM 参数限制内存

## 💰 费用说明

微信云托管按量计费，预估费用：

- **CPU（0.5核）**：约 0.055元/小时
- **内存（1GB）**：约 0.032元/小时
- **合计**：约 0.087元/小时 ≈ **2元/天** ≈ **60元/月**

**省钱技巧：**
- 不用的时候可以停止服务（不收费）
- 新用户有免费额度
- 可以设置自动休眠（无访问时自动停止）
